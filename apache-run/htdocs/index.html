Welcome to a Digital Public Library of America Vertical Search Demo created for the DPLA Beta Sprint by the California Digital Library

Why explore a web crawler approach?

  • No need to rationalize and ingest metadata
  • Leverage curation expertise of libraries to focus enterprise search technology
  • Builds on existing technology we can easily deploy

Technology used

This demo is built using Apache Nutch 1.3 for web crawling and LucidWorks Certified Distribution for Solr Release 3.2 for search with a simple web interface using django templates. The Cloudera Distrobution of Hadoop will be run on Amazon EC2 Spot Instances to create the index for the demo. The goal is to do minimal programming for the demo, focusing on configuration of the crawl and search. Code and configuration files are on Google Code.

Targeted content

IMLS Digital Collections and Content is a registry of digital materials funded by IMLS National Leadership Grants and selected LSTA-supported collections. The IMLS DCC will provide the foundation for the demo index. In addition, resources from the University of California and digital resources such as the Nineenth-century Scholarship Online Federated Websites will be in the initial crawl.

The full seed list is maintained in Google Code. If you have ideas for additional resources that should be included in the demo or a later phase please let us know and suggest a site.

Integrating mass digitized monograph collections such as the Hathi Trust will not be investigated in this demo for both technical and content development reasons. While we believe that a crawler approach could be used to aggregate monographs, digitized monographs do not have the same granularity and characteristics as web pages and short documents often found on the web. The out-of-the-box stack we are using is tuned for web crawling and search, and we anticipate book search may not be optimized by default. Furthermore, we believe that access to digitized local history and cultural history is an import role of libraries and this library developed content should be made more visible.

How it could be used / built out

Future phases of the project, if pursued, could put programming effort into developing plugins or filters for Nutch or Solr that would add data to the records necessary to build more search facets. Perhaps linked data embedded in content pages or machine learning techniques could be employed to enhance the search experience.
